Actividad 1.a

Explica las diferencias entre un desarrollador front-end, back-end y full-stack.


Puede decirse que un desarrollador front-end es el que se ocupa del desarrollo en el entorno del cliente, osea, programa la parte visual de la web usando HTML, CSS y JavaScript , el desarrollador back-end se ocupa de la parte del servidor, usando lenguajes como PHP, NodeJS (Javascript) etc y el desarrollador full-stack es el que desarrolla el proyecto en ambos escenarios

Actividad 1.b

¿Qué es un framework CSS? ¿Qué es Bootstrap? ¿Cuántos más framework CSS existen y cuáles son?


Un framework CSS es un conjunto de código y librerías para facilitar la labor a los desarrolladores a la hora de programar código ya que suele usarse para ahorrar tiempo, obteniendo resultados finales muy llamativos.

Bootstrap es un claro ejemplo de FrameWork CSS. Se usa para crear sitios más responsivos y está orientado a los dispositivos móviles y no hace falta tener un gran conocimiento de HTML5 para poder usarlo.

Aparte de Bootstrap, existen los siguientes Frameworks CSS

  1. TailWind CSS
  2. Foundation
  3. Bulma
  4. Pure.CSS
  5. Milligram
  6. Ulkit

Actividad 1.c

¿Qué es WCAG? ¿Por qué es tan importante validar en XHTML, CSS y WCAG 2.1 AAA?


WCAG Son una serie de directrices para indicar como se debe diseñar una web para facilitar el acceso a cualquier persona que acceda a ella. Es importante realizar dicha validación para asegurarnos que nuestra web se vea bien en cualquier dispositivo, ya sea un pc de sobremesa o un teléfono móvil.

Actividad 1.d

Explica con tus propias palabras qué es un servidor web, qué es el protocolo HTTP, busca ejemplos donde se ejecuten los métodos HTTP GET y POST y explica qué es Apache y qué es XAMPP.


Un servidor web (como Apache 2) es un software que procesa la aplicación programada del lado servidor realizando conexiones con un cliente, usando el protocolo HTTP, y mostrándole a éste el resultado de dicho procesamiento.


El protocolo HTTP o de transferencia de hypertexto es un protocolo de aplicación ideado para facilitar la comunicación entre sistemas distribuidos en ubicaciones remotas (Aunque sirve también para usos locales)


Un fácil y claro ejemplo de uso de GET lo encontramos directamente en el buscador de Google. Sabemos que usa el metodo get ya que los términos de la búsqueda aparecen en la barra de direcciones.


Los lugares donde usuamente usan POST para tratar los datos de un formulario son los típicos paneles en los que introducimos nuestros datos de acceso (o registro) para, por ejemplo, cualquier red social como Facebook o Linkedin. Sabemos que usan POST ya que no muestran ningún dato en la barra de direcciones debido a que son datos "sensibles" y/o privados
