Actividad 5.a

¿Qué es un control de versiones? ¿Qué diferencias hay entre uno centralizado y otro distribuido?


El control de versiones es un sistema que registra los cambios realizados sobre un archivo o conjunto de archivos a lo largo del tiempo de tal manera que sea posible recuperar versiones especificas más adelante.

Un sistema centralizado es aquel en el que todas las copias se registran en un servidor central, mientras que en el distribuido, las copias pueden distribuirse libremente entre los nodos, existiendo así varias copias distribuidas en la red.

Actividad 5.b

¿Qué es Git? ¿Y GitHub? ¿En qué se diferencian y cómo se relacionan entre sí?


Git es un sistema de control de versiones distribuido, lo que significa que un clon local del proyecto es un repositorio de control de versiones completo. Estos repositorios locales plenamente funcionales permiten trabajar sin conexión o de forma remota con facilidad.

GitHub es una de las principales plataformas para crear proyectos abiertos de herramientas y aplicaciones, y se caracteriza sobre todo por sus funciones colaborativas que ayudan a que todos puedan aportar su granito de arena para mejorar el código. Como buen repositorio, el código de los proyectos que sean abiertos puede ser descargado y revisado por cualquier usuario, lo que ayuda a mejorar el producto y crear ramificaciones a partir de él. Y si prefieres que tu código no se vea, también pueden crearse proyectos privados. Se relacionan en que GitHub es una plataforma basada en la web que incorpora las características de control de versiones de Git.
